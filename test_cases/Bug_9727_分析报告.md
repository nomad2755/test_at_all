## 问题描述
申请的界面可以使用登录的短信验证码，绕过了申请页面的独立验证。

## 可能原因分析

| 可能原因 | 概率 | 说明 |
|--------|------|------|
| **验证逻辑混用** | 高 | 短信验证码验证接口被申请和登录两个场景混用，没有区分场景 |
| **权限打通** | 中 | 登录验证通过后，自动赋予了申请权限，绕过了单独验证 |
| **Session 复用** | 中 | 登录成功后复用了申请页的验证状态 |
| **接口未区分** | 中 | 后端 API 没有区分 `scene` 参数 |

## 建议检查点

```
1. 检查后端验证逻辑 - 是否有场景区分
2. 检查登录和申请的 token 是否独立
3. 检查权限校验 - 是否有自动授权逻辑
4. 检查 UI - 是否可以绕过
```

## 修复建议

1. 在后端验证接口中添加 `scene` 参数区分场景
2. 为申请和登录使用独立的验证服务
3. 增加场景特定的权限校验
4. 前端增加场景参数传递
