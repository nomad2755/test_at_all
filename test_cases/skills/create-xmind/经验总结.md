**问题**：用户需要XMind格式方便管理，同时需要HTML格式查看图片。

## 2026-04-17 XMind文件修复

### 问题描述

生成的XMind文件无法正常打开，用XMind 2021打开时报错或空白。

### 根本原因

1. **文件名拼写错误**：`meatdata.xml` → 正确为 `metadata.xml`
2. **缺少必要文件**：少了 `comments.xml`、`meta.xml`、`META-INF/manifest.xml`
3. **终端输出乱码**：Windows终端GBK编码导致print中文显示乱码（但文件本身UTF-8正确）

### 正确文件结构

```
.xmind (ZIP压缩包)
├── content.xml              # 测试用例内容（必须）
├── styles.xml               # 样式定义（必须）
├── comments.xml              # 注释文件（必须）
├── meta.xml                 # 元数据文件（必须）
└── META-INF/
    └── manifest.xml          # 清单文件（必须）
```

### 修复代码

```python
def create_xmind():
    xmind_path = "测试用例.xmind"
    with zipfile.ZipFile(xmind_path, 'w', zipfile.ZIP_DEFLATED) as zf:
        # 核心文件（5个必须）
        zf.writestr('content.xml', make_content_xml())
        zf.writestr('styles.xml', make_styles_xml())
        zf.writestr('comments.xml', make_comments_xml())  # 必须
        zf.writestr('meta.xml', make_meta_xml())          # 必须
        zf.writestr('META-INF/manifest.xml', make_manifest_xml())  # 必须

    # 验证
    with zipfile.ZipFile(xmind_path, 'r') as z:
        bad = z.testzip()  # None = 正常
        content = z.read('content.xml').decode('utf-8')
        print('商标列表展示' in content)  # True = 中文正确
```

### 注意事项

1. **文件名不要拼错**：特别是 `metadata.xml` 不是 `meatdata.xml`
2. **5个文件缺一不可**：content.xml + styles.xml + comments.xml + meta.xml + META-INF/manifest.xml
3. **终端乱码不影响文件**：Python文件写入UTF-8编码正确，Windows终端显示乱码是正常的
4. **用XMind 2021打开**：不是XMind 2024
